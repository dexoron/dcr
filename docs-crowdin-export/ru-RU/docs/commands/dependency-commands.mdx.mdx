# Команды зависимостей

## `dcr add \<name> [source]`

Добавляет зависимость в `dcr.toml`.

```bash
dcr add fmt                    # поиск в реестре
dcr add mylib ../path/to/lib   # путь как источник
dcr add mylib path:./lib       # явный префикс пути
dcr add mylib git:https://github.com/user/repo  # явный префикс git
dcr add mylib github:user/repo # сокращённая запись GitHub
dcr add mylib gitlab:user/repo # сокращённая запись GitLab
```

Префиксы источника:

* `path:` — локальный путь
* `git:` — URL адрес git-репозитория
* `github:` — расширение `https://github.com/\<user>/\<repo>`
* `gitlab:` — расширение `https://gitlab.com/\<user>/\<repo>`
* `http://` / `https://` / `git@` — полный URL

Флаги:

* `--branch \<name>` — git ветка
* `--tag \<name>` — git tag
* `--rev \<hash>` — git commit

Если источник не указан — DCR выполняет поиск в подключенных реестрах.

## `dcr tree`

Показывает дерево зависимостей проекта.

```bash
dcr tree
```

Пример вывода:

```
my-app v0.1.0
├── fmt (registry)
│   └── spdlog (registry)
└── catch2 (registry)
```

В случае path зависимостей, рекурсивно отображает их собственные зависимости (из файла `dcr.toml`).
